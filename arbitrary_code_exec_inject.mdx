💉 Arbitrary Code Injection & Execution aka ACI/ACE ⚙️ 

---

#Arbitrary Meaning  🔍 

Based on random choice or personal whim, rather than any reason or system.

---



#ACI  💉 

Injection means to insert code or commands in a machine or a process .
It occurs when an application or a process evaluates inputs without validating it first

---
#ACE  ⚙️ 

Its an attacker's ability to execute arbitrary commands or code on a target machine or a target process  by using an exploit.
For the purpose of process/machine behaviors changes.

#Causes 🤷🏻‍♂️

- Input not checked properly <sub></sub>
- Input not sanitized properly <sub></sub>
- Language native Functions <sub><sup>(fprintf, printf, sprintf, setproctitle, syslog )</sup></sub>
- OS native Libraries <sub><sup>(WIN API => Kernel32.dll, User32.dll and GDI32.dll)</sup></sub>
- eval()<sub><sup>(Unsafe & Dangerous function in PHP & JS)</sup></sub>
---

Different Type Injection 
- SQL Injection <sub><sup>(Insert arbitrary SQL commands in database queries)</sup></sub>
- OS Command Injection <sub><sup>(Insert arbitrary commands in SHELL calls)</sup></sub>
- LDAP Injection <sub><sup>(Alter the logic of an LDAP Query)</sup></sub>
- XPath Injection <sub><sup>(Alter the logic of an XPath search)</sup></sub>
- DLL Injection  <sub><sup>(Alter behavior of a program by using external programs)</sup></sub>
- Buffer overflow <sub><sup>(	Alters the flow of an application by overwriting parts of memory.)</sup></sub>

There is no secrets ✨ to prevent ACE & ACI :
 - Manage your buffer and memory allocation with security rules 
 - Check & Control your method & functions access
 - Check your data type, values 
 - Restrict Data & Native functions to your final user
 - Dont use unsafe methods/functions
 - Use prepared Queries
